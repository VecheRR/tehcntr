# Деплой на PHP-хостинг

PHP для этого сайта не нужен. Сайт работает как статический фронтенд: HTML, CSS, изображения и клиентский JavaScript.

## Что передать разработчикам или администратору

1. Установить зависимости локально:

```bash
npm install
```

2. Собрать сайт:

```bash
npm run build
```

3. Залить на сервер содержимое папки `dist`, а не весь проект.

В корень сайта должны попасть файлы из `dist`: `index.html`, папка `assets`, папка с публичными изображениями, `icon.svg`, `.htaccess`.

## Если на сервере Apache

Файл `.htaccess` уже добавлен в проект и после сборки попадает в `dist`. Он нужен, чтобы прямые ссылки вроде `/solutions` и `/technologies` открывались корректно.

## Если сервер просит PHP-файл

Можно переименовать `dist/index.html` в `index.php`. PHP-код добавлять не нужно, содержимое файла менять не требуется.

## Что не нужно делать

- Не загружать папку `src` на сервер.
- Не загружать `node_modules`.
- Не запускать `npm install` или `npm run build` на PHP-хостинге.
- Не переписывать сайт на PHP, если не нужны формы с отправкой на сервер, база данных или личный кабинет.
