# Техноцентр - лендинг

Современная версия лендинга на Vite, React, TypeScript и Framer Motion.

## Стек

- Vite
- React
- TypeScript
- Framer Motion
- CSS без внешнего UI-фреймворка

## Как запустить локально

```bash
npm install
npm run dev
```

После запуска Vite покажет локальный адрес, обычно:

```bash
http://localhost:5173/
```

## Как собрать для сервера

```bash
npm run build
```

Готовая статическая сборка появится в папке:

```bash
dist
```

Именно содержимое папки `dist` нужно размещать на сервере или статическом хостинге.

## Как проверить production-сборку локально

```bash
npm run preview
```

## Где менять контент

Основной контент сайта находится в:

```bash
src/App.tsx
```

Там сгруппированы:

- пункты меню;
- отрасли;
- решения;
- партнеры и ссылки;
- технологические блоки;
- контакты.

Страницы `Решения` и `Технологии` открываются через hash-маршруты `/#/solutions` и `/#/technologies`. Это сделано специально для простого PHP/статического хостинга: при обновлении страницы браузер всегда загружает `index.html`, а нужный раздел восстанавливается из адреса.

## Где менять стили

Все стили находятся в:

```bash
src/styles.css
```

Основные цвета заданы в `:root`:

```css
--green: #48a770;
--green-dark: #2f7f53;
--gray-900: #333333;
```

## Где лежат изображения и логотипы

Публичные изображения лежат в:

```bash
public/assets
```

В коде они подключаются через путь вида:

```tsx
/assets/technocenter-logo.png
```

Если нужно добавить новый логотип, положите файл в `public/assets` или `public/assets/partners`, затем укажите путь в `src/App.tsx`.

## Передача разработчикам через GitHub

1. Создайте новый репозиторий на GitHub.
2. Загрузите в него весь проект, включая `package.json`, `src`, `public`, `README.md`.
3. Разработчики запускают:

```bash
npm install
npm run dev
```

4. Для публикации они выполняют:

```bash
npm run build
```

5. На сервер отправляется содержимое папки `dist`.
